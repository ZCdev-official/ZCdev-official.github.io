Hi there, I'm Zach.

ZCdev is the name I (Zachary Cowan) use for developing. This developer name provides an easily recognizable way for people to identify me as the creator of some code or design. I started to teach myself coding somewhere around 7th or 8th grade after taking a basic class, and I've never looked back since. I had always been good at math, so it only made sense to start coding. However, I didn't stop with coding. I had always wanted to make a game, and games had good artwork. As a result, I began to learn how to do graphic design. This combination of knowing how to code and design has given me a distinct advantage whenever I approach a project.

“Making the simple complicated is commonplace; making the complicated simple, awesomely simple, that’s creativity.”

Charles Mingus

Charles Mingus makes an excellent point in this quote, and I've come to embrace it. Simplicity is key. When you make something complex, very few people can understand it. But when you take that complex thing and make it simple, everyone can understand it and enjoy it. As a result, I try to embrace this idea of simplicity whenever I approach a project.

Code

Coding is a language for math. With it, developers can do some very creative things. And just like any other language, coding is used to communicate with people.

Design

Although coding is used to communicate with people, it is usually too complicated for anyone to care about it. Graphical designs allow developers to communicate their code in a much simpler way.

Publish

Coding and designing are great. However, nobody cares unless they are able to see the final product. As a result, it is crucial for developers to publish their work so that others may enjoy it.
